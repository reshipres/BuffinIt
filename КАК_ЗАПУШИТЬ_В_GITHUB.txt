🚀 СПОСОБ 1: Автоматический (Python скрипт)

1. Дважды кликните на файл: push_to_github.py
   
   ИЛИ запустите в терминале:
   python push_to_github.py

2. Следуйте инструкциям на экране

3. Если попросит авторизацию - введите:
   - Username: ваш_логин_github
   - Password: токен (получить на GitHub → Settings → 
              Developer settings → Personal access tokens)


─────────────────────────────────────────────────────────────

🔧 СПОСОБ 2: Вручную через Git Bash / Terminal

Откройте Git Bash в папке проекта и выполните:

1. Инициализация (если нужно):
   git init

2. Добавление remote:
   git remote add origin https://github.com/reshipres/BuffinIt.git

3. Добавление файлов:
   git add .

4. Коммит:
   git commit -m "BUFF Pay Bot - Initial commit"

5. Установка ветки:
   git branch -M main

6. Push:
   git push -u origin main

   Если репозиторий не пустой (есть README):
   git push -u origin main --force


─────────────────────────────────────────────────────────────

🔑 АВТОРИЗАЦИЯ В GITHUB

При первом push Git попросит авторизацию:

Username: ваш_логин_github
Password: ЛИЧНЫЙ ТОКЕН (не пароль!)

Как получить токен:
1. GitHub.com → Settings
2. Developer settings
3. Personal access tokens → Tokens (classic)
4. Generate new token
5. Выберите срок действия и права (repo)
6. Скопируйте токен (покажется только раз!)
7. Используйте вместо пароля

• Используйте личный токен вместо пароля
• Если репозиторий не пустой - используйте --force


─────────────────────────────────────────────────────────────

🆘 ПРОБЛЕМЫ И РЕШЕНИЯ:

1. "fatal: remote origin already exists"
   Решение: git remote remove origin

2. "error: failed to push"
   Решение: git push -u origin main --force

3. "Authentication failed"
   Решение: Используйте личный токен, не пароль

4. "Permission denied"
   Решение: Проверьте права доступа к репозиторию
